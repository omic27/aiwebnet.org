# Как ставить задачи Codex (шаблоны)

## Универсальный префикс к любой задаче
Перед началом:
1) Прочитай docs/PRODUCT.md
2) Прочитай docs/ARCHITECTURE.md
3) Прочитай docs/ANALYTICS_LOGIC.md

Ограничения:
- не ломать архитектуру
- не переименовывать файлы/папки
- минимальные изменения
- бот всегда отвечает пользователю (даже при ошибках)

Формат результата:
- план (какие файлы, что меняешь)
- изменения
- список изменённых файлов
- как проверить (команды)

## Пример задачи: аудит и фиксы ответов
"Найди места, где бот не отвечает пользователю или падает при отсутствии данных. Исправь минимально. Добавь обработку ошибок и единый fallback-ответ SKIP."
